# 布局规则

渲染前应用以下规则。优先保证稳定、清晰，而不是一味压缩图形尺寸。

## 通用规则

- 将第一个顶层内容放在接近 (20, 20) 的位置。虽然支持负坐标，但通常没有必要使用。
- 主流程默认从左向右排列。
- 节点边界之间保留 80～120px 水平间距；同级节点之间至少保留 56px 垂直间距。
- 线性连接的节点按视觉中心对齐。
- 长标签优先增加节点宽度，并同步更新右侧、顶部和底部端口偏移。应优先精简文字，不要缩小字号。
- 为连线标签预留空间，避免把节点放在可能经过的正交线路径上。
- 相关组件放在同一父容器中。不要把属于泳道或架构层的节点直接放在根容器下。

## 分支布局

- 将判断节点放在主流程轴线上。
- 主要分支或肯定分支继续向右排列。
- 次要分支或否定分支放在判断节点下方。
- 如果汇聚后的路线容易混淆，使用独立的结束节点区分不同结果。
- 分支标签保持简短且语义完整。

## 水平泳道

- 泳池标题高度使用 36px，泳道标签列宽使用 96px。
- 第一条泳道从 `pool.y + 36` 开始。
- 每条泳道高度至少为 140px，节点位置满足 `x >= pool.x + 96`。
- 条件允许时，节点与泳道边界之间至少保留 32px。
- 通过泳道归属表达责任主体；连线跨越泳道属于正常情况。

## 垂直泳道

- 泳池标题高度使用 36px，泳道标签行高使用 32px。
- 每条泳道宽度至少为 220px。
- 节点放在 `lane.y + 32` 以下。
- 只有阅读方向或参与者数量更适合列布局时才使用垂直泳道。

## 架构分层

- 除非用户指定其他方式，否则架构层垂直堆叠，层间距保持 40～60px。
- 架构层标题高度使用 38px，内部边距使用 20px。
- 层内节点放在 `layer.y + 58` 以下。
- 架构层尺寸至少为 760×180；空间不足时优先增加宽度，不要挤压组件。
- 跨层且垂直关系明确时使用 `top`/`bottom` 端口；其他情况使用 `right`/`left`。

## 图片检查

渲染后出现以下任何问题时，应修改场景并重新生成：

- 关键标签被裁剪或省略；
- 节点或容器发生重叠；
- 节点超出所属泳道、分组或架构层；
- 连线标签覆盖节点；
- 路径暗示了错误的流程方向；
- 无法区分不同分支的结果；
- 孤立坐标导致画布出现大量无意义空白。
